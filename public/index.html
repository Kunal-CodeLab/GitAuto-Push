How to Generate a GitHub PAT (Token)

A Personal Access Token (PAT) allows this automation tool to securely push your code to GitHub without triggering username/password credential prompt popups, especially when managing multiple GitHub accounts.

  1. Login to your account on GitHub.com.
  2. Click on your profile picture in the top-right corner and go to Settings.
  3. Scroll down in the left sidebar and click on Developer Settings.
  4. Select Personal Access Tokens → Tokens (classic).
  5. Click Generate new token → Generate new token (classic).
  6. Enter a note (e.g. GitAutoPush Tool) and select the expiration.
  7. Check the box for repo (Full control of private repositories). This is required!
  8. Scroll to the bottom and click Generate token.
  9. Copy the token immediately! (It starts with ghp_). You will not be able to see it again.
  10. Paste the token into the input field in this tool.
Security Note: This token is processed entirely on your local machine and is never sent to any external server (except GitHub itself during push authentication).
